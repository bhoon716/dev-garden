# OAuth 2.0

## 한 줄 정의

외부 서비스나 인증 제공자를 통해 권한 위임 또는 로그인 정보를 얻는 표준 프로토콜이다.

## 핵심 개념

- OAuth 2.0은 사용자의 비밀번호를 직접 받지 않고 제한된 권한을 위임받는다.
- OIDC는 OAuth 2.0 위에 인증 계층을 얹어 ID Token으로 사용자 신원을 전달한다.
- Authorization Code Flow가 서버 기반 웹 애플리케이션에서 주로 사용된다.

## 사용 예

- Google Login
- GitHub OAuth Login

## 주의할 점

- OAuth는 인가 프레임워크이고, OIDC가 로그인 인증 정보를 표준화한다.
